# Component Card Template

每个组件使用下面这张卡，保证字段稳定：

```markdown
### CXX. 组件名称

- 最小技术问题：
- 组件内容：
  - 
  - 
- 输入：
- 输出：
- 关键假设 / 约束：
- 为什么需要它：
- 依赖组件：
- 被哪些组件使用：
- 证据锚点：
- 最小可替代方案：
- 关键消融 / 诊断建议：
  - 
  - 
- 可复用抽象：
- 与已有组块的连接：
```

约束：

- 组件名称优先用“动词 + 名词”
- `证据锚点` 优先写 Section / Figure / Table / Eq / Algorithm / Appendix
- `可复用抽象` 写成“问题类型 ↔ 技术手段”
